# コードスタイルガイド

プロジェクト全体で統一されたコード品質を保つためのガイドラインです。

## 言語とコメント

- **言語**: 日本語
  - コード内のコメント、ドキュメント、コミットメッセージはすべて日本語で記述してください。
- **命名規則**: 英語
  - 変数名、関数名、ファイル名などは英語を使用し、内容がわかる説明的な名前を推奨します。
- **絵文字**: 原則禁止
  - コミットメッセージや通常のコメントでの絵文字使用は避けてください。
  - 代替として `*強調*` や `【注記】` などのテキスト表記を使用します。

## コミットメッセージ規約

Prefixをつけて変更内容を明確にします。

| Prefix | 説明 | 例 |
| :--- | :--- | :--- |
| `fix` | バグ修正 | `fix: タイムラインのレイアウト崩れを修正` |
| `hotfix` | クリティカルなバグ修正 | `hotfix: 起動時にクラッシュする問題を修正` |
| `add` | 新規機能ファイル追加 | `add: 設定画面の実装` |
| `update` | 機能修正（バグ以外） | `update: 文言の変更` |
| `change` | 仕様変更 | `change: アニメーションの対象を変更` |
| `clean` | 整理リファクタリング | `clean: 未使用変数の削除` |
| `disable`| 機能無効化 | `disable: リップルエフェクトを無効化` |
| `remove` | ファイル機能削除 | `remove: 旧アイコン画像を削除` |
| `upgrade`| バージョンアップ | `upgrade: ライブラリの更新` |
| `revert` | 変更の取り消し | `revert: 前回コミットを取り消し` |

## JavaScript コーディング規約

- **構文**: モダンなJavaScript (ES6+) を使用してください。
- **フォーマット**:
  - インデント: スペース2つ
  - セミコロン: 常に使用する
  - 文字列: シングルクォート `'` を優先 (HTML文字列などはバッククォート `` ` `` も可)
- **非同期処理**: `Promise` チェーンよりも `async/await` を推奨します（可能であれば）。

## CSS コーディング規約 (拡張機能)

- **命名**: BEM記法に似た、意味のあるクラス名を使用してください (例: `.config-card__header`)。
- **Scoped**: 他のサイトに影響を与えないよう、特有のプレフィックス (例: `.xmo-`) をつけるか、Shadow DOMの使用を検討してください（今回はPopupがメインなので影響は限定的）。
